# Universal Product Rules

These rules apply to every seek-and-find book, regardless of theme, audience, or format.

## Product Promise

Every book must have a clear promise:
- who it is for
- what kind of hunt it offers
- what makes it visually appealing
- what makes it different from generic hidden-object books

## Commercial Appeal

Every book should be designed to look appealing:
- in Amazon search thumbnails
- on a product detail page
- as a gift
- in preview images
- when printed in hand

## Audience Fit

Every book must define a target audience before page creation begins.

Required audience fields:
- primary age range
- expected difficulty
- acceptable visual complexity
- acceptable humor level
- safety/tone limits

## Replay Value

A good seek-and-find page should reward repeat viewing.

Include:
- secondary hidden objects
- small visual jokes
- micro-scenes
- recurring motifs
- optional bonus challenges

Avoid:
- one-note pages
- empty filler
- repetitive object placement
- identical scene structures across the book

## Differentiation

The book must not rely on being a clone of a famous property.

It should differentiate through:
- original mascot
- original theme
- original world
- original art style
- original object lists
- original page structure

## Brand Consistency

If part of a series, the book must maintain:
- consistent title format
- consistent mascot design
- consistent difficulty promise
- consistent art quality
- consistent page rules
- consistent audience targeting

## Buyer Trust

The book must deliver what the cover and listing promise. Do not advertise:
- more pages than included
- coloring pages if pages are not colorable
- age ranges that the actual difficulty does not support
- hidden objects that are not actually present

## Rejection Triggers

A page or book should be rejected if it:
- imitates a copyrighted character or trade dress
- has unfair hiding
- is too muddy to solve
- is not age-appropriate
- has inconsistent mascot design in a mascot series
- depends on unreadable tiny details
- looks low-effort or AI-broken

# Universal Art Rules

These rules apply to all generated or commissioned seek-and-find illustrations.

## Readability Comes First

A seek-and-find page can be dense, but it must remain readable.

Required:
- clear silhouettes
- separated shapes
- recognizable objects
- intentional detail clusters
- balanced contrast
- distinct foreground, midground, and background

Avoid:
- visual mud
- random clutter
- over-rendered tiny details
- low-contrast object soup
- too many similar objects near the target list items

## Composition

Each page should have:
- a strong main scene concept
- visual flow through the page
- at least three meaningful zones
- no large dead areas unless intentionally reserved for title/layout
- no overloaded single corner while other areas are empty

## Detail Distribution

Detail should be distributed across the whole page.

Rules:
- no more than one zone may feel noticeably underdeveloped
- hidden objects should not all appear in the foreground
- important details should not be crammed into trim edges
- dense areas should have enough contrast to remain solvable

## Visual Hierarchy

Even puzzle pages need hierarchy.

The viewer should be able to understand:
1. what kind of place this is
2. where the main activity is happening
3. what smaller details are worth exploring

## Character Design

Characters must be:
- expressive
- readable at page size
- appropriate for the audience
- consistent within the book
- visually distinct from protected characters

## Color

Color should support searchability.

Rules:
- use contrast intentionally
- do not hide important objects only by making them nearly invisible
- maintain appealing palette harmony
- avoid muddy dark-on-dark or light-on-light hiding unless for adult difficulty and still fair

## Print-Aware Detail

Fine details must survive print.

Avoid:
- ultra-thin critical lines
- important tiny facial expressions
- hidden objects that only work when zoomed in
- objects placed where binding or trim may remove them

## Thumbnail Appeal

At thumbnail size, the page or cover should still show:
- strong theme
- lively composition
- attractive color
- clear product category

# Universal Hiding Rules

These rules define fair and satisfying hidden-object placement.

## Fairness Standard

A hidden object is fair when:
- it is present
- it is identifiable once found
- it is not destroyed by cropping, blur, or distortion
- it is not hidden only by being too small
- it is not hidden only by being placed on the page edge
- it is supported by the target list or visual clue system

## Difficulty Sources

Good hiding may use:
- partial cover
- similar colors
- busy but readable surroundings
- playful pose
- object integration into the scene
- visual misdirection
- scale variation
- thematic camouflage

Bad hiding relies on:
- invisibility
- extreme tiny size
- near-zero contrast
- ambiguous shapes
- placement under trim
- distorted or incomplete object forms
- fake duplicates that make the answer unclear

## Target Visibility

Once found, a target should trigger the reaction:
> Oh, there it is!

Not:
> I guess that blob is supposed to be it?

## Placement Distribution

Hidden targets should be distributed across the full page.

Default distribution:
- foreground: no more than 30% of listed targets
- midground: 40–50% of listed targets
- background: 20–30% of listed targets

Project overlays may adjust this, but crowding all objects in one area is forbidden.

## Edge Rule

Do not place primary targets on the extreme page edge.

Secondary objects may be near an edge only if:
- they are not at risk of trim
- they remain identifiable
- they are not part of a repeated cheap trick

## Primary Mascot Hiding

For mascot-based books:
- mascot appears once unless project rules say otherwise
- mascot should not be in the same type of location on every page
- mascot should not always be central, foreground, or edge-hidden
- mascot must retain identifiable signature features
- mascot must be visible enough to feel fair

## Duplicate Confusion

Do not include accidental duplicates of hidden target objects unless:
- the puzzle intentionally says there are multiple
- only one has a specific required variation
- the difference is clear for the target audience

## Count Reliability

Image models may fail exact counts. Verify all target items manually.

## Difficulty Labels

Use the difficulty scale from `17_difficulty_scale.md`:
- Level 1: very easy
- Level 2: easy
- Level 3: moderate
- Level 4: challenging
- Level 5: expert

Each page brief should state intended difficulty.

# Universal Print and KDP Rules

These rules keep pages usable for print-on-demand production.

## Format Planning

Before creating pages, define:
- trim size
- bleed or no-bleed
- color or black-and-white interior
- paperback or hardcover
- page count target
- whether pages are single-page puzzles or two-page spreads

## Safe Area

Important search targets should not be placed:
- inside trim risk zones
- too close to the gutter
- under page numbers
- under planned text/list areas
- at the extreme edge

## Resolution

Final art must be high-resolution enough for print.

Recommended:
- 300 DPI at final trim size
- larger working files when possible
- avoid upscaling tiny low-quality images as final pages

## Print Contrast

Printed pages often look darker and less vibrant than screen previews.

Check:
- dark areas do not swallow hidden objects
- tiny objects are not lost
- color-only distinctions are not too subtle
- coloring-book lines remain clean

## Text Policy

Interior seek-image puzzle art should not contain accidental text, gibberish letters, fake labels, watermarks, or speech bubbles unless a specific project requires controlled text.

For seek-image generation, prompts should usually include:
- no text
- no labels
- no speech bubbles
- no watermark
- no page number
- no border unless intentionally needed

For non-seek page images that are designed to carry words, such as title pages, story/list pages, recap pages, closing pages, and instruction pages, prompts must request the exact approved generated text. Do not add a no-text constraint to these prompts. Instead, forbid only extra words, misspellings, fake text, gibberish, watermarks, and unapproved labels.

Title-page prompts are not flexible title concepts. They must request the exact approved title words only. For mascot title pages, reference the character images saved in ChatGPT Sources by exact uploaded name and include the written mascot description from the active project canon. Unresolved placeholders and generic book labels are hard failures. If the exact title, required source images, or written mascot description are missing, stop and ask instead of generating.

## Hidden Object List Placement

If target lists are added later in layout, leave enough visual or margin space. Do not bake lists into image art unless the workflow specifically requires it.

## Bleed Warning

If using full-bleed pages, keep all required objects inside the safe area. Decorative background may extend to bleed.

## Gutter Warning

For two-page spreads:
- do not hide important objects across the gutter
- do not place mascot face or critical details in the binding fold
- ensure the page still works if slightly misaligned

## Coloring Books

For seek-and-color mode:
- line art must print cleanly
- important objects must be distinguishable without color
- avoid gray shading that makes coloring unpleasant
- leave enough open space for coloring

# Universal Legal and Safety Rules

These rules apply to every project.

## Copyright and Trademark Safety

Do not imitate or closely resemble:
- Where's Waldo / Where's Wally
- famous hidden-object franchises
- protected cartoon characters
- recognizable branded mascots
- trademarked outfits, logos, symbols, or trade dress
- distinctive copyrighted worlds

## Waldo-Adjacent Avoidance

Forbidden as a focal design:
- red-and-white striped shirt as the primary character identity
- bobble hat paired with round glasses and a search-mascot role
- direct parody that still functions as a market substitute
- names or titles that invite direct confusion with Waldo/Wally

## Originality Requirement

Every series should have:
- original mascot or object system
- original theme framing
- original page concepts
- original title
- original visual identity

## Child Safety

For children’s books:
- no sexualized content
- no graphic violence
- no realistic gore
- no hidden disturbing imagery
- no hateful symbols
- no unsafe instructional content
- no frightening content beyond the selected audience tolerance

## Adult Content

Adult seek-and-find books may use mature humor or darker themes only if:
- the product is clearly labeled for adults
- the content is legal and non-exploitative
- the cover/listing does not mislead parents
- the content does not violate platform rules

## Cultural and Historical Themes

If using real cultures, religions, historical groups, or mythology:
- avoid caricature
- avoid lazy stereotypes
- use respectful visual treatment
- separate fantasy invention from real-world representation

## Medical, Legal, or Safety Claims

Do not make unsupported claims in title, subtitle, cover, or listing copy.

## AI Disclosure and Platform Compliance

Follow the current publishing platform requirements for AI-generated content disclosure. Check the platform rules before upload because requirements can change.

## Legal Review Trigger

Escalate for review if a page, mascot, title, or cover:
- feels recognizably similar to an existing property
- uses a real brand/logo
- contains celebrity likeness
- uses protected symbols
- targets children with questionable content

# Universal Quality Bar

This file defines the minimum professional standard.

## Page Quality

Every page must:
- communicate a clear theme
- provide a fair search challenge
- contain intentional visual detail
- avoid empty filler zones
- look polished enough for sale
- support the target audience
- pass print-risk review

## Book Quality

Every book must:
- have a clear concept
- maintain consistent art direction
- vary page settings enough to prevent boredom
- follow a difficulty plan
- have consistent hidden-object rules
- avoid obvious AI errors
- deliver its product promise

## Series Quality

Every series must:
- have a recognizable identity
- use repeatable mascot or motif rules
- support multiple future themes/pages
- remain legally distinct
- stay consistent without becoming repetitive

## Instant Rejection Criteria

Reject or revise any page with:
- missing primary target
- hidden objects impossible to identify
- mascot inconsistency severe enough to confuse readers
- broken anatomy that distracts from the puzzle
- unreadable clutter
- accidental text/gibberish
- copyrighted resemblance
- major print-safety risk
- obvious AI artifacts in important areas

## Good Page Test

A strong page should make the viewer want to:
1. look closer
2. find the target
3. explore the side details
4. come back later

## Parent/Buyer Test

For children’s books, a buyer should immediately understand:
- the age range
- the theme
- the activity
- why the book looks fun
- why it feels giftable

## Adult Buyer Test

For adult books, a buyer should immediately understand:
- whether it is relaxing, funny, hard, cozy, weird, nostalgic, or challenging
- whether it is a coloring book or full-color book
- whether the difficulty matches the listing promise

# Target List Rules

These rules govern hidden-object lists.

## Target List Purpose

The target list gives the reader a clear reason to explore the page. It should support the theme and difficulty, not feel random.

## Object Selection

Good hidden objects are:
- recognizable
- theme-appropriate
- visually distinct
- fun to find
- not overly similar to many background objects
- suited to the target age range

Weak hidden objects are:
- too generic
- impossible to distinguish
- tiny and shapeless
- culturally confusing for the audience
- too similar to surrounding clutter
- difficult to draw consistently

## Count Guidance

Default counts:
- ages 3–5: 5–8 objects
- ages 5–8: 10–15 objects
- ages 8–12: 15–25 objects
- teens/adults: 20–50 objects depending on page density
- seek-and-color: reduce count if line art becomes cramped

These are defaults, not laws. Project overlays may adjust.

## Variety Rule

A target list should include a mix of:
- shapes
- sizes
- colors
- categories
- difficulty levels

Example fantasy list categories:
- treasure
- food
- magical tools
- creatures
- clothing/accessories
- symbols
- everyday objects with fantasy flavor

## Avoid Duplicate Confusion

If the list includes "cupcake," do not scatter many similar cupcakes unless the target asks for a specific one.

Better:
- blue cupcake with star sprinkles
- cracked dragon egg
- silver key with heart-shaped bow

## Object Naming

Use simple, age-appropriate names.

For younger children:
- "gold key" is better than "ornate brass skeleton key"

For adults:
- more specific names can increase challenge and interest

## Bonus Objects

Bonus objects may be included, but do not let them obscure the main task.

Examples:
- Find 5 tiny mushrooms.
- Count all the cats.
- Spot the object that appears on every page.

## Verification

After generation, verify:
- every listed object exists
- object count is correct
- each object is identifiable
- no object is hidden in an unfair way
- no accidental duplicate makes the answer ambiguous

# Difficulty Scale

Use this scale to define and review puzzle difficulty.

## Level 1 — Very Easy

Best for:
- ages 3–5
- first page of a children's book
- warm-up puzzles

Traits:
- large targets
- high contrast
- minimal occlusion
- simple scenes
- few objects

## Level 2 — Easy

Best for:
- ages 4–6
- simple children’s books
- early pages

Traits:
- clear targets
- light camouflage
- moderate scene detail
- few misleading objects

## Level 3 — Moderate

Best for:
- ages 5–8
- general children’s seek-and-find
- most commercially friendly kid pages

Traits:
- busy but organized scenes
- partial cover allowed
- similar-color hiding allowed
- targets still identifiable once noticed
- typical solve time around 30–90 seconds for the primary mascot

## Level 4 — Challenging

Best for:
- ages 8–12
- older children
- family puzzle books
- light adult books

Traits:
- denser scenes
- smaller targets
- stronger camouflage
- more visual misdirection
- still fair without zooming

## Level 5 — Expert

Best for:
- adults
- advanced seek-and-find
- deliberately hard puzzle books

Traits:
- high density
- subtle camouflage
- complex target lists
- longer solve times
- requires careful scan, but not unfair invisibility

## Difficulty Planning

A book should usually vary difficulty.

Recommended children's book pattern:
- early pages: Level 2–3
- middle pages: Level 3
- late pages: Level 3–4
- final challenge page: Level 4 if audience supports it

Recommended adult book pattern:
- early pages: Level 3–4
- main pages: Level 4
- bonus challenge pages: Level 5

## Difficulty Warning

Harder does not mean messier. A messy page is not a difficult page; it is a broken page wearing a fake mustache.

# Children General Audience Rules

These rules apply to all children’s seek-and-find projects.

## Tone

Children’s projects should feel:
- safe
- playful
- warm
- curious
- visually inviting
- emotionally clear

## Content Limits

Avoid:
- graphic violence
- realistic horror
- sexual content
- cruel humiliation
- hateful symbols
- unsafe instructions
- disturbing hidden imagery
- content that would surprise or upset a parent buying for the stated age

## Visual Clarity

Children need clear object recognition.

Rules:
- use recognizable object shapes
- avoid overly abstract targets
- keep important objects distinguishable
- do not rely on tiny details that require adult-level visual patience

## Humor

Good child-friendly humor:
- silly animals
- harmless chaos
- funny costumes
- visual mix-ups
- gentle slapstick
- whimsical surprises

Avoid:
- mean-spirited jokes
- jokes based on adult references
- gross-out content beyond the project’s stated tolerance

## Difficulty

Challenge should create satisfaction, not frustration.

Default:
- make primary target findable in a reasonable time
- include easier and harder secondary objects
- avoid hiding every object at maximum difficulty

## Buyer Appeal

Children’s pages should look giftable. Parents should see the page and think:
- this is fun
- this is safe
- this will keep a child occupied
- this looks polished

# Standard Seek-and-Find Mode

Use this mode for full-color or grayscale seek-and-find books that are not primarily coloring books.

## Main Task

The reader searches for:
- one mascot
- a list of hidden objects
- or both

## Art Style

Art may be:
- full-color
- grayscale
- painterly
- cartoon
- detailed line-and-color
- collage-inspired if print-safe

## Core Requirements

Each page must:
- have a clear scene
- include all listed targets
- use fair hiding
- remain readable at print size
- avoid accidental text or artifacts
- match the selected audience overlay

## Hidden Object List

Target lists may be:
- printed on the opposite page
- placed below the image
- included in a separate answer/key section
- represented with small icon references if layout supports it

Do not bake target-list text into generated art unless intentionally required.

## Answer Key

For books with moderate or higher difficulty, include an answer key plan.

Answer keys may use:
- circled reduced images
- coordinates
- object-by-object notes
- page thumbnails

## Revision Priorities

When revising standard pages, prioritize:
1. missing/malformed primary target
2. unfair hiding
3. unreadable clutter
4. print risks
5. theme enhancement

# Seek-and-Find Coloring Book Mode

Use this mode when the book must function as both a puzzle and a coloring book.

## Core Difference

A coloring seek-and-find page must be solveable before coloring and enjoyable to color after solving.

## Art Style

Required:
- black-and-white line art
- clean outlines
- printable contrast
- no muddy gray rendering
- enough open spaces to color
- hidden objects recognizable by outline alone

## Detail Density

Coloring pages cannot be as dense as full-color seek-and-find pages.

Rules:
- reduce object count
- preserve open coloring areas
- avoid excessive tiny texture
- avoid dark filled-in areas
- use line weight hierarchy

## Hidden Object Rules

Hidden objects must:
- remain identifiable without color
- not depend on color clues
- not be buried in texture
- be large enough to color if appropriate

## Object Count Defaults

Recommended:
- ages 3–5: 4–6 objects
- ages 5–8: 8–12 objects
- ages 8–12: 12–18 objects
- adults: 15–30 objects, depending on intricacy

## Prompt Requirements

Seek-and-color puzzle prompts should specify:
- coloring book page
- crisp black-and-white line art
- no grayscale shading
- clean outlines
- white background or printable page background
- hidden objects integrated into the line art
- no generated text unless the page is a non-seek text page with exact approved wording

## QA Priority

A seek-and-color page fails if it is:
- too cramped to color
- too faint to print
- dependent on color to solve
- full of muddy shading
- visually unpleasant as a coloring page

- color palette
- signature clothing/accessory
- facial/personality traits
- size relative to world
- prohibited changes

## Consistency Rule

Every page prompt must restate mascot design details unless using an approved reference system.

## Primary Target Rule

Default:
- mascot appears once per puzzle page
- mascot is hidden fairly
- mascot signature features remain visible once found

Project overlays may define exceptions.

## Hiding Variety

Do not hide the mascot the same way every time.

Vary:
- pose
- scene zone
- partial cover type
- surrounding colors
- activity
- distance from focal center

Avoid:
- always behind a tree
- always in a window
- always in the bottom corner
- always wearing different colors
- changing species/proportions

## Mascot Recognition

The mascot should remain recognizable by:
- silhouette
- color
- accessory
- face
- body shape

## Cover Use

The mascot should usually be clearly visible on the cover, even if hidden inside the pages.

## Failure Triggers

Reject or revise if:
- mascot is missing
- mascot looks like a different character
- signature accessory is absent when required
- mascot is too hidden to identify
- mascot resembles a protected character

# Non-Mascot Hidden Object Mode

Use this mode when the book does not have one recurring hidden character.

## Main Task

The reader searches for a target list, object category, count challenge, or puzzle clue.

## Advantages

This mode works well for:
- adult puzzle books
- coloring books
- themed object hunts
- educational seek-and-find
- holiday books
- cozy scenes

## Requirements

Each page must have:
- a clear target list or task
- enough variety in object types
- a balanced difficulty level
- a reason the objects belong in the scene

## Object Recurrence

Recurring objects may appear across pages, but they are not mascots unless positioned as recurring characters.

## Difficulty Control

Without a mascot, difficulty is controlled by:
- object count
- object size
- visual similarity
- scene density
- specificity of target names
- bonus challenges

## Failure Triggers

Reject or revise if:
- target list feels random
- too many objects are ambiguous
- scene lacks a clear concept
- objects are hidden unfairly

# Canonical Ember Project Overlay and Character Canon

This Ember section is active source-of-truth content inside the four-file bundle. It replaces the old archived `02_EMBER_ACTIVE_RULESET.md`, `06_EMBER_CHARACTER_CANON.md`, and legacy Twizzleflame/Ember files for active production decisions.

## Ember Active Structure

Active Ember interiors use the **10-spread story + seek** model:

| Printed page(s) | Role |
|:---:|:---|
| 1 | Title page |
| 2 / 3 | Story/list page 1 + seek image 1 |
| 4 / 5 | Story/list page 2 + seek image 2 |
| 6 / 7 | Story/list page 3 + seek image 3 |
| 8 / 9 | Story/list page 4 + seek image 4 |
| 10 / 11 | Story/list page 5 + seek image 5 |
| 12 / 13 | Story/list page 6 + seek image 6 |
| 14 / 15 | Story/list page 7 + seek image 7 |
| 16 / 17 | Story/list page 8 + seek image 8 |
| 18 / 19 | Story/list page 9 + seek image 9 |
| 20 / 21 | Story/list page 10 + seek image 10 |
| 22-23 | Two-page mission recap / reward spread showing the 10 mission items |
| 24 | Closing + next-book teaser |

If another front-matter page is added before the first story/list page, remap the spreads first. The rule is always: **produce the seek page first, then produce its paired story/list page**.

Default production order when story starts on page 2:

```text
1, 3, 2, 5, 4, 7, 6, 9, 8, 11, 10, 13, 12, 15, 14, 17, 16, 19, 18, 21, 20, 22-23, 24
```

## Ember Canon

Reference names:

- `Ember-001`
- `Ember-002`
- `Ember-003`

Core identity:

- Ember is the recurring baby dragon guide and mascot.
- He is warm, curious, brave, and helpful.
- He is visible in seek scenes as the child's helper, never the hidden target.
- He is small, expressive, and child-friendly.

Required Ember description for prompts:

```text
Ember is a tiny adorable reddish-orange baby dragon guide, about small-cat sized, with soft pumpkin-orange and coral shading, a cream belly, tiny shiny golden spiral-comma horns, large glossy blue-teal eyes, a plain bright blue-teal scarf, and a tiny plain brown crossbody satchel with dull-gold button clasps. He has rounded plush-like baby-dragon proportions, soft storybook scale texture, bright friendly eyes, and a cheerful curious expression. He is not a fox, cat, human wizard, or generic fantasy animal.
```

Visual QA rules:

- Match the required Ember description above.
- Do not simplify Ember into a fox, cat, human wizard, generic fantasy animal, or off-model mascot.
- No scary, smug, angry, or overly realistic expression.
- No costume that resembles copyrighted characters.

Pose/use rules:

- On seek-image pages, show Ember exactly once as a guide/helper in foreground or midground.
- On story/list pages, Ember may appear as a small vignette character.
- Any pointing, gaze, tail direction, body lean, or gesture on a story/list page should lead the viewer left-to-right toward the facing seek page.
- Do not make Ember so large that he crowds out the seek-and-find content unless it is cover/title/closing art.
- Hide the mission item and other findable objects around the scene, not on Ember.
- Avoid making Ember dominate a seek page like cover art.

## Supporting Cast

Use supporting characters only when they are called for by a page prompt. Keep them original, simple, and non-branded.

Approved recurring roles:

- Hootiepuff — fuzzy owl clue-spotter
- Pebblekins — shy rock-dragon friend
- Luma Leafwhisk — enchanted forest sprite friend
- Gemma Glint — pink crystal castle guide
- Elder Glowkeeper — wise golden dragon elder

Avoid:

- villain-coded characters
- scary monsters
- copyrighted character silhouettes
- overly detailed recurring side characters unless the user approves them as canon

Only add side-character references if the page actually includes that character. If a needed reference file is not present or the name is uncertain, stop and ask the user before producing a final prompt.

Default Ember reference line:

```text
Use the Ember reference images named in the Ember Canon section for Ember's baby dragon appearance and proportions.
```

## Ember Page Standards

Seek image pages are the right-side full-color puzzle scenes. Each seek image page must contain:

- one full-color seek-and-find scene
- Ember visible as guide/helper, not hidden
- one special mission item hidden fairly
- broad-category clutter in the prompt, not a fixed final checklist
- no readable text, labels, checklist, arrows, circles, page numbers, watermarks, speech bubbles, or answer marks
- print-safe composition with important objects away from trim/gutter risk

After seek art approval:

1. Ask chat to identify **20-30 candidate finds** visible in the seek image.
2. Human checks the list against the image.
3. Select **1 mission item**, **10 main printed finds**, and **5-15 bonus finds**.
4. Save a `.md` source file for the facing story/list page.
5. Generate the story/list page with exact text and themed icons.
6. Verify spelling, line breaks, icons, and object names.

Story/list pages are left-side pages. They are designed story pages, not worksheets. Each story/list page must include:

- one short heading
- prewritten 2-4 sentence story prose specific to that book/location
- a mission line asking the child to help Ember find the special mission item
- a checklist area with the mission item first
- **1 mission item**, listed first
- **10 main printed finds** selected from the approved facing seek image
- **5-15 bonus finds** selected from the approved facing seek image
- themed icons first; icons may be simplified/stylized but must match clearly enough for kids
- exact text from the `.md` source file, with no extra words or misspellings

External fallback: if themed icons cause too many errors, switch to fancy checkboxes. If exact generated text fails after reasonable attempts, generate story/list art without text and add text in layout.

All story/list pages share one underlay format:

- vertical 8.5 x 11 page, 17:22 ratio, full-color, print-safe margins
- warm cream/parchment storybook background with soft magical texture
- consistent top heading band
- consistent story text box sized for 2-4 short sentences
- mission line treatment stronger than body text
- consistent checklist container with themed icon slots
- small Ember/world vignette area, usually left or lower-left
- calm, low-clutter supporting illustration with open breathing room around text and list areas
- decorative details should support the page theme without competing with the printed find list
- any character pointing, gaze, gesture, lean, or framing directs attention **left-to-right** toward the facing seek page
- same rounded, child-readable font target across all story/list pages

Allowed variation: book/location motifs, accent colors, border decorations, checklist container shape, small character pose, and minor placement shifts that preserve reading order: **heading -> story -> mission line -> checklist**.

## Ember Text and Recap Rules

Seek-image prompts must not generate readable text.

Title-page prompts must reference the Ember character images named in the Ember Canon section unless the uploaded source names differ. They must also include the required written Ember description from the Ember Canon section. Title-page prompts must use exact approved title words only. Unresolved placeholders and generic book labels are hard failures. If the exact approved title, required source images, or written Ember description are missing, stop and ask instead of generating.

Story/list pages, title pages, recap pages, instruction pages, and closing pages must request exact generated text from the approved source text when using ChatGPT Image 2. The prompt must supply the exact words and layout, and must forbid extra words, misspellings, fake text, gibberish, unapproved labels, and watermarks. Verify every word. If text fails after review, regenerate without text and add it in layout.

Pages 22-23 show the 10 special mission items for that book as a celebratory reward spread. This is not a per-scene answer key and does not mark where small objects were hidden.

Traditional location-style answer keys are optional and not required. If used later, keep them as separate files, PDFs, web pages, or QR-linked hints.

## Ember Core Visual Style

Use a polished soft 2.25D children's storybook seek-and-find illustration style with lightly illustrated edges, rounded plush-like forms, gentle sculpted volume, painterly shading, warm cheerful lighting, mostly smooth clean surfaces, minimal visible outlines, subtle pencil-like edge details, magical but clean and readable, not flat cartoon, not realistic, and not overly 3D or plastic-looking.

Strict avoid list: no Waldo-style red-and-white striped focal outfit, Disney/HP/Waldo clones, copyrighted character imitation, scary mood, weapons, photorealism, anime style, flat vector style, muddy clutter, tiny invisible objects, generated labels on seek pages, arrows, circles, answer marks, page numbers, or watermarks.
