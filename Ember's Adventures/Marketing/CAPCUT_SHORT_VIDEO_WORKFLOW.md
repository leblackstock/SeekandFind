# CapCut Short Video Workflow

Project: Ember and the Sparkleflame Festival Search

Purpose: create no-voice short videos from existing promotional images for Pinterest-adjacent social platforms:

- YouTube Shorts
- TikTok
- Instagram Reels
- Facebook Reels

This workflow is documentation only unless video creation is explicitly requested.

## Core Rules

- Use CapCut Online free tools only.
- Do not start a paid trial.
- Do not click upgrade, pro, export paid-only, or other paid options.
- Do not post directly to any platform.
- Export and download MP4 files only.
- Use 9:16 vertical format.
- Target duration: 6 to 12 seconds, usually 8 seconds.
- Export 1080p MP4 if free export allows.
- Use simple slow zoom/pan animation.
- Basic animations and transitions are acceptable when they appear to be free.
- Basic music appears to be free; use it only when it is clearly free.
- If unsure whether a music track is free or commercially safe, skip music.
- Do not use AI voice, talking avatar, narration, or paid/pro effects.
- If a required action is blocked by a paywall, stop and report the blocker.

## Successful Base Workflow

1. Open CapCut Online.
2. Create a new vertical 9:16 project.
3. Upload or select the source promotional image.
4. Before uploading or adding the image to the timeline, set the timeline/playhead position to `0:00`.
5. Upload or add the image only after the playhead is at `0:00`.
6. Place the image on the timeline.
7. Set the image/video duration, usually 8 seconds.
8. Apply a simple free animation:
   - slow zoom in, slow zoom out, gentle pan, or basic transition
   - keep Ember and image text visible
   - do not crop off key text
9. Add on-screen text only when requested.
10. Format the text before moving on.
11. Move timeline/playhead pointer to the beginning.
12. Add free/basic music only if clearly available.
13. Trim/cut music to match the exact video duration.
14. Confirm every active track starts at `0:00` and extends to the same final endpoint.
15. Run QA.
16. Export/download as MP4.
17. Record completion status before starting the next repeat.

## Text Rules

If text is added, fully format and position it before moving on:

- readable at phone size
- no typos
- not covering Ember's face
- not covering important title/image text
- no "available now" until the book is live
- no extra words
- do not simply repeat the main words already printed in the source image
- use overlay text as a short hook, invitation, or added context

If the source image already says the same thing as the planned overlay, rewrite the overlay before exporting. Example from Short #2: the image already says "MEET EMBER / YOUR BABY DRAGON GUIDE," so the overlay was changed from repeating that idea to "Ready for a tiny festival quest?"

Use subtle readability help when white text sits on busy artwork:

- first choice: subtle dark text shadow
- acceptable if needed: light stroke or simple background
- avoid heavy sticker-like styling unless requested

Layer order matters. Put the text layer above the image/video layer in the timeline; otherwise the overlay can be present on the timeline but invisible on the canvas.

## Music Rules

- If music is added, move the timeline/playhead pointer to the very beginning of the video before adding music.
- After adding music, trim/cut the music so it ends exactly at the chosen video length.
- If the free/commercial status is unclear, skip music.

## Known Issues And Fixes

Known issue: when music was added, CapCut inserted it at the current timeline/playhead position instead of the start.

Fix: always move the timeline/playhead pointer to `0:00` before adding music, then trim the music to the final video length.

Known issue: when a new image was uploaded/added before the timeline/playhead position was set to `0:00`, CapCut inserted the image at the current timeline position instead of the start.

Fix: always set the timeline/playhead position to `0:00` before uploading, selecting, or adding the replacement image. Do this before the file-upload step, not after. If the image lands late anyway, delete the misplaced image and repeat the upload from `0:00` instead of trying to clean up a bad placement.

Known issue: a newly inserted still image can default to about 5 seconds even when the template is about 8 seconds.

Fix: extend the right edge of the image clip to match the chosen video length and align with the text/music end.

Known issue: after replacing media, tracks can look close but still have tiny gaps at the beginning or end. This can create a black start or black tail in the export.

Fix: before export, check every active track on the timeline:

- image/video starts at `0:00`
- text starts at `0:00`
- music starts at `0:00`
- image/video ends at the same final endpoint as text and music
- text ends at the same final endpoint as image/video and music
- music ends at the same final endpoint as image/video and text
- start-frame check: click `0:00` and confirm the image is visible
- end-frame check: click just before the final endpoint and confirm the image is still visible

Do not export if any track begins late, ends early, or creates a black start/end.

Known issue: during cleanup, the text layer can end up below the image layer.

Fix: move the text layer above the image layer before QA.

Known issue: music may need to be rechecked after timeline cleanup.

Fix: confirm the music track still starts at `0:00`, ends with the video, and remains on an audio track after moving or deleting clips.

## Export And Download Flow

Use this free download path:

1. Click `Export`.
2. Choose `Download`; do not choose TikTok, schedule, share for review, or presentation.
3. In export settings, confirm:
   - Resolution: `1080p`
   - Quality: `Recommended quality`
   - Frame rate: `30fps`
   - Format: `MP4`
4. Rename the export in the `Name` field to the final short filename without `.mp4`.
5. Click the export button in the settings panel.
6. Wait for `Exported`.
7. Click the centered video `Download` button.
8. Do not click the desktop-app download prompt.
9. Save the MP4 into this marketing folder with the final filename.
10. Verify the saved MP4 before updating the tracker.

## QA Checklist

- 9:16 vertical
- correct duration
- image visible and not badly cropped
- Ember visible when relevant
- title/image text readable when relevant
- overlay text readable
- overlay text has no typos
- overlay text adds something beyond the source image headline
- text shadow/stroke/background used if needed for readability
- text layer is above the image/video layer
- all active tracks start at `0:00`
- all active tracks end at the same final endpoint
- start-frame check shows the image, not black
- end-frame check shows the image, not black
- no "available now"
- no paid/pro elements used
- no accidental extra text
- music starts at beginning if used
- music ends at video end
- no direct posting happened
- export settings checked: 1080p, 30fps, MP4
- MP4 exported/downloaded
- final filename recorded
